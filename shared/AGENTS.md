# shared 目录协作说明

本文件适用于 `shared/` 目录及其子目录。

## 目录职责

- 本目录放置前后端共享的常量、契约、规范化函数和轻量级公共工具
- 这里的代码会被多端消费，因此变更要格外注意兼容性

## 编写原则

- 优先写“纯函数 + 常量”
- 避免引入运行环境副作用，不要依赖 DOM、浏览器对象、Express 上下文或数据库连接
- 尽量不要从 `server/` 或 `src/` 反向导入；`shared/` 应保持底层、稳定、可复用

## 契约变更要求

- `shared/contracts/` 中的函数应负责：
  - 输入兜底
  - 类型归一化
  - 返回结构稳定
- 修改响应字段、默认值或限制常量时，必须考虑前端与后端是否都需要同步调整
- 能兼容旧数据时优先兼容，不要轻易做破坏式修改

## 风格建议

- 函数命名优先使用：
  - `normalize*`
  - `sanitize*`
  - `resolve*`
- 保持实现简洁、可测试、无隐藏副作用
- 不在这里堆积业务编排逻辑；共享层负责“规则定义”，不是“流程 orchestration”

## 验证建议

- 对 `shared/` 的改动，优先补或改 `tests/contracts/` 相关测试
- 如果某个共享函数被多个模块依赖，修改后至少检查一个前端调用点和一个后端调用点
